# Roadmap — the-one-scenario-corpus

Continuation lines for the project after the initial version. Benchmark goal: a scenario corpus with **no linear correlation** between scenarios for studying OppNet routing protocols (see [analysis/reports/observaciones_correlacion.md](analysis/reports/observaciones_correlacion.md)).

**Diagnosis:** The issue is not statistical (low power with n=d=33) but **geometric**: many scenarios lie in a small subspace (same map, same WDM+HelsinkiMedium model, same structure). Varying only TTL, rate or message size does not change the dynamic graph structure → high correlation.

**Strategy:** Do not add more scenarios of the same style; **differentiate** existing ones and add a few **radically different** ones (spatial structure, dynamic regime, resource regime).

**Current state:** **corpus_v1** contains **70** scenarios. Steps 4.1 (clustering), 4.2 (diversification) and 4.3 (~10 radical scenarios) are done. Next priority: bilingual documentation (section 1).

## 3. Corpus diversity: criteria and goals

**Diversity criteria (not only |r|):**
- **|r| < 0.7** for ≥95% of pairs (or 100% if required).
- **Minimum cosine distance > 0.05** (no almost-identical pair).
- **No pair with cos_dist < 0.05**.
- **Silhouette > 0.3** if clustering is used (well-separated spaces).

**Realistic goals after diversification:** max |r| < 0.85; pairs with |r| ≥ 0.7 **< 3%**; min cos_dist > 0.05.

**Three axes to break correlation:** (1) Spatial structure: different map, clusters, partition, mule, sparse vs dense graph. (2) Dynamic regime: stationary, periodic, burst, phase change. (3) Resource regime: extremes (buffer 256KB vs 200MB, range 5m vs 200m, speed 0.2 vs 15 m/s).

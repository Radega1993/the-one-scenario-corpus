## Overview

**the-one-scenario-corpus** is a **scenario corpus and analysis pipeline** for the [The ONE](https://akeranen.github.io/the-one/) simulator (Opportunistic Network Environment). It provides:

- **70 simulation scenarios** (`.settings` files) organised in 7 families
- **Feature extraction, correlation analysis, clustering, and output-based validation**
- **An interactive dashboard** to explore results

The goal is to offer a **reproducible benchmark** with **diverse, non-redundant scenarios** for evaluating routing protocols in opportunistic networks (DTN/OppNets)—e.g. for thesis work or papers.

---

## Why this corpus exists

- Routing protocols need to be tested on **varied** scenarios; if all scenarios are linearly similar, results are redundant.
- We build a **feature vector** per scenario from its `.settings` and check **correlation** and **distance** between scenarios so the corpus is **diverse** and useful for benchmarking.
- Criteria: low linear correlation (|r| < 0.7), minimum cosine distance between scenarios, and (optionally) clustering and output-based analysis.

| **Cosine distance (min)** | 0.0534 (target > 0.05) |

**Diversity criteria:** Currently **not** fully met: 93.7% of pairs have |r| < 0.7 (target ≥95%). Minimum cosine distance is above 0.05 (no almost-identical pairs). Work is ongoing to diversify the most correlated pairs.

## How to get started

1. **Clone** the repository (and ensure The ONE simulator is built).
2. Follow **[Quickstart](Quickstart)** to run scenarios and the analysis pipeline.
3. Open the **dashboard**: `streamlit run scenarios/analysis/dashboard.py`

For full documentation, see the [README](https://github.com/Radega1993/the-one-scenario-corpus/blob/main/README.md) in the repo.
